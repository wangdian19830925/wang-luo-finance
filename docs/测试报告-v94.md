## 4. Bug 发现与修复

### 4.1 已修复 Bug

| Bug | 等级 | 描述 | 根因 | 修复方案 | 验证 |
|-----|------|------|------|---------|------|
| BUG-14 | 🔴 P0 | v93 部署后线上仍显示 v92 | GitHub Pages Jekyll 构建失败，CDN 不回源 | 添加 `.nojekyll` 禁用 Jekyll，重新部署 v94 | ✅ 线上已显示 v94 |
| BUG-15 | 🔴 P0 | iPhone 饼图图例文字被截断 | v93 使用 `text-overflow: ellipsis` | 移除 ellipsis，改为左对齐 + 文字换行 | ✅ 自动化测试通过，待用户 iPhone 验证 |
| BUG-16 | 🟠 P1 | `deploy_github_api.py` 中文文件路径 404 | 未对中文字符进行 URL 编码 | 使用 `urllib.parse.quote(path, safe='/')` 编码 | ✅ 已重新部署成功 |
| BUG-17 | 🟡 P2 | `__pycache__` 被提交到 git | 缺少 `.gitignore` 规则 | 添加 `__pycache__/`、`*.pyc`、`*.pyo` 并删除已提交文件 | ✅ 已清理 |

### 4.2 修复过程

1. **v92 版本号问题**：通过 `curl` 检查线上 `index.html` 发现版本仍为 v92，同时 GitHub Pages build status 为 `errored`。添加 `.nojekyll` 后重新触发构建，状态变为 `built`，版本更新为 v94。
2. **饼图截断问题**：v93 对 `.pie-legend-label` 使用 `text-overflow: ellipsis`，导致文字被截断。v94 移除 `white-space: nowrap`/`overflow: hidden`/`text-overflow: ellipsis`，改为 `word-break: break-word` + `overflow-wrap: break-word`，并将图例布局从右对齐改为标准左对齐（圆点在左，文字在右）。
3. **中文路径编码问题**：部署 docs 目录下的中文文件名时 `urllib.request` 抛出 `UnicodeEncodeError`。修复 `api_request` 和 `get_file_sha` 中对路径使用 `urllib.parse.quote(..., safe='/')`。
