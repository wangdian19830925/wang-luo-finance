# 测试报告 v204

## 版本概述

v204 对基金数据模型进行了根本性重构，以份额（shares）为锚定字段，持仓金额（holdValue）= 份额 × 净值 为派生计算；允许同一代码基金分多条录入；基金趋势图按代码分组配对展示。同时修复了公积金余额保存/读取的参数类型和字段名匹配问题。

## 变更内容

1. **基金数据模型重构（份额锚定）**
   - `saveFund()`：份额为必填字段（toast 错误提示），holdValue = shares × nav 自动计算
   - `editFundPrice()`：shares > 0 → holdValue = shares × nav；shares = 0（旧数据）→ shares = holdValue / nav
   - `_applyPriceData()` 基金部分：简化为 shares-anchor 逻辑，navDerived 仅保留为数据标记
   - `importFundData()`：模板更新只改 name/currency/market/nav，不覆盖 shares/holdValue/costValue

2. **同一代码多条基金记录**
   - 基金 ID 从 `code` 改为自动生成（`Date.now().toString(36) + random`）
   - `_getStableBusinessKey()` 对基金返回 `null`（原来返回 `code`），解除业务键去重限制
   - `importFundData()` 同代码模板更新从 `findIndex` 改为 `filter`（多条记录全量更新）
   - `_fetchLiveFundPricesOnly()` 添加 `uniqueCodes` 去重，避免重复 API 调用

3. **基金趋势图按代码分组**
   - `loadFundList()` 重写为分组显示：按代码分组，每组显示记录卡片 + 汇总 + 趋势图占位
   - `renderFundTrend()` 重写为按代码独立趋势图，每组一条聚合 holdValue 曲线
   - `index.html` 移除固定 `fundTrendSection`，趋势图动态插入各组内

4. **公积金余额修复**
   - `saveInlineEdit()` 公积金部分：从传对象改为传两个数值参数
   - `loadProvidentFundBalance()`：字段名 `params.monthly` → `params.providentFundMonthly`

## 测试结果

**全量 316 断言通过**（31 单元 + 181 集成 + 14 房贷 + 27 房贷趋势图 + 21 退休曲线 + 42 冒烟）

- FUND-NAV 01~05：holdValue = shares × nav 正确计算；shares 保持不变；旧数据反算正确
- FUND-MULTI 01~04：同一 code 两条记录共存且 ID 不同；NAV 刷新各条独立；删除一条不影响另一条；基金业务键为 null
- PROVIDENT-FUND 01~04：余额和月缴分别保存/读取；更新一个不影响另一个
- IMPORT-GUARD 01~08：导入不覆盖用户修改字段；模板字段正常更新
- IMPORT-LWW 01~05：模板更新 skipUpdatedAt 保护生效；LWW 正确保留用户数据
