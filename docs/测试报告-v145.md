## 本次变更

1. 修复 CloudBase 推送逻辑导致云端 `data` 字段丢失/未写入的问题：
   - 对已有文档优先使用 `doc.update(docData)`；
   - `update` 失败时回退到 `doc.set(docData)`；
   - 文档不存在时仍使用 `set()` 新建。
2. 增加推送后主动验证：
   - 写入成功后立即 `get()` 回读云端文档；
   - 检查 `data` 字段是否存在、记录总数是否大于 0；
   - 若验证异常，在控制台输出明确警告，便于后续排查。
3. 同步修正 `index.html` 中 `storage.js` 的 query string（此前停留在 `?v=135`），避免浏览器/Service Worker 缓存旧版 storage.js。
4. 提升 `clientVersion` 到 `v145`，所有版本号保持一致。

## 根因说明

v144 将推送从 `set(docData, { merge: true })` 改为 `set(docData)` 覆盖写，但实际在 mac/Safari 端仍出现“推送成功但云端文档仍只有 `_passwordHash`/`_passwordEnabled`，没有 `data`”的现象。

由于 v139 及之前使用 `doc.update(docData)` 时财务数据同步正常，而 `set()` 在 CloudBase v2 JS SDK 上多次表现异常，因此 v145 改回以 `update()` 为主、新建时才用 `set()` 的策略，并加入回退与验证。
