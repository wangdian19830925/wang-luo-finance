## 变更摘要

1. **优化 `pushToCloud()` 策略**
   - 去掉每次推送前的激进 `docRef.remove()` 删除操作，避免删除-创建之间的竞态和缓存问题。
   - 先 `docRef.get()` 探测文档是否存在；存在则 `update()`，不存在则 `create()`。
   - `create()` 失败时（如文档已存在）才回退到删除重建。

2. **推送后验证改为 where 查询**
   - 不再使用 `docRef.get()` 做推送后验证（该接口在 CloudBase 中似乎存在缓存/副本延迟）。
   - 统一使用 `collection.where({ docId: 'finance_data' }).get()` 回退验证，与 `pullFromCloud()` 的可靠路径一致。
   - 等待时间从 300ms 增加到 400ms。

3. **日志脱敏改进**
   - 所有 `passwordHash` 日志输出改为 `len=xx,head=xxxx` 格式，避免 "exists" 被误解为实际值。

4. **修复版本号不一致**
   - `index.html` 中 `storage.js`/`app.js`/`style.css` 查询参数从 `?v=148` 修正为 `?v=150`，避免浏览器/SW 实际加载旧版 JS。
   - `tests/smoke_test.js` 版本断言同步修正为 v150。

5. **版本统一升级**
   - `index.html`、`service-worker.js`、`js/storage.js`、`tests/smoke_test.js` 全部升级到 v150。
