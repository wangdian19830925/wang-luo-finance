## 修改内容
1. **将密码 hash/enabled 完全嵌入 `jsonData` 字符串内部**
   - 根因：CloudBase 后端对独立字段 `passwordHash` / `_passwordHash` 做过滤/替换，v147-v151 无论怎么改，云端最终都被替换为 `"len=N,head=..."` 等脱敏字符串，导致刷新或跨设备后密码丢失。
   - 修复：`pushToCloud()` 不再推送任何独立密码字段；`_getLocalDataPackage()` 将本地 `finance_password_hash` / `finance_password_enabled` 存入 `data._authHash` / `data._authEnabled`，随 `jsonData` 整体序列化到云端；`pullFromCloud()` 解析 `jsonData` 后读取 `_authHash` / `_authEnabled`。

2. **修复 `_getPasswordFields()` 未解析 `jsonData` 的隐患**
   - 根因：`pushToCloud()` 推送后验证阶段，从 `where` 查询返回的 `verifyDoc` 只含 `jsonData` 字符串，`doc.data` 未解析，`_getPasswordFields()` 读不到 `_authHash`，可能误判 `pwdMatch: false`。
   - 修复：在 `_getPasswordFields()` 中增加 fallback：当 `doc.data` 中未找到 `_authHash` 时，尝试 `JSON.parse(doc.jsonData)` 读取 `parsed._authHash` / `parsed._authEnabled`。

3. **版本统一升级到 v152**
   - `index.html`、`service-worker.js`、`js/storage.js`、`tests/smoke_test.js`。
