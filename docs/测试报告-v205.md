## 变更内容

### 核心逻辑重写：saveFund()
- **必填字段**：基金代码、投入成本(costValue)、当前持仓金额(holdValue)（移除份额/净值/名称的手动填写）
- **流程**：
  1. 用户输入基金代码 → blur 事件触发 JSONP 查询东方财富 API → 自动填充基金名称
  2. 用户输入投入成本和当前持仓金额
  3. 点击"查询并保存"按钮 → 系统通过天天基金 API 获取最新净值(nav) → 计算 `份额 = 持仓金额 / 净值`
  4. API 查询失败时仍保存（nav=0, shares=0），toast 提示"请稍后刷新净值"
  5. 后续净值刷新：`holdValue = 份额 × 新净值`（份额是锚定不变的字段）

### 表单变更（index.html）
- 移除：名称(fundName)、份额(fundShares)、净值(fundNav) 的手动输入框
- 保留：代码(fundCode)、投入成本(fundCostValue)、当前持仓金额(fundHoldValue) — 均为必填（红色*标注）
- 名称改为只读显示框(id="fundNameDisplay")，由 API 自动填充
- 提交按钮文案改为"查询并保存"（查询期间按钮禁用防重复提交）
- 取消按钮移到保存按钮之后

### setupForms 变更（app.js）
- fundCode 输入框添加 blur 事件监听器，触发 `_autofillFundName()`

### _applyPriceData 基金段变更
- NAV 刷新逻辑简化：shares>0 → `holdValue = shares × newNav`；shares=0 → `shares = holdValue / newNav` 反算

### editFundPrice 变更
- 简化为 shares 锚定逻辑：shares>0 → `holdValue = shares × nav`；shares=0 → `shares = holdValue / nav` 反算
