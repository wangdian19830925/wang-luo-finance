## 本次变更

1. **修复 CloudBase 推送后云端数据为空的问题**
   - 将云端数据存储方式改为 `jsonData`（JSON 字符串），避免 CloudBase v2 JS SDK 对嵌套对象/数组的写入异常；
   - 保留对旧 `data` 字段的读取兼容，确保已有历史数据可迁移。
2. **推送逻辑改为始终使用 `set()` 覆盖写**
   - 移除 `update()` 主路径，避免已有损坏文档导致字段丢失；
   - `set()` 后立刻 `get()` 回读验证。
3. **增加损坏文档删除重建机制**
   - 若推送后验证发现云端数据为空，自动删除该文档并重新 `set()` 创建；
   - 防止 `finance_data` 文档进入不可修复的损坏状态。
4. **版本号统一升级到 v146**
   - `index.html` / `service-worker.js` / `js/storage.js` / `tests/smoke_test.js` 全部同步。

## 根因说明

v145 日志显示：`doc.update()` 返回成功（`updated: 1`），但随后 `get()` 回读发现文档为空，且原有 `_passwordHash`/`_passwordEnabled` 也变为 `undefined`。这表明云端 `finance_data` 文档已处于损坏状态，或 CloudBase v2 SDK 对当前文档的嵌套对象写入存在异常。通过将数据序列化为 JSON 字符串并使用 `set()` 覆盖写，可绕过嵌套对象写入问题；若仍失败，则删除重建文档。
