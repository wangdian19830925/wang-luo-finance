## 根因说明

根据日志分析，v145-v147 的云端 `finance_data` 文档出现字段值被保存为字符串 `"null"` 与 `"N/A"`，且 `set`/`create` 返回成功但验证时 `passwordHash` 不存在。这表明业务字段在写入或读取阶段被 SDK / 数据库规则过滤。最可能的触发点是字段名以 `_` 开头：在 MongoDB 及 CloudBase 文档数据库中，`_id`、`_openid` 等均为系统保留字段，业务字段以下划线开头可能被安全网关或 SDK 静默丢弃，而返回的成功信息仅说明文档操作完成，不代表所有字段均被接受。去掉下划线前缀后，字段将被视为普通业务字段，绕过该限制。
