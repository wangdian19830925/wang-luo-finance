## 修复内容

### 关闭密码保护时需要验证密码（安全漏洞修复）

**根因：** `togglePassword()` 函数直接切换 `cfg.enabled` 标志，关闭密码保护时不要求输入当前密码。任何人拿到设备就能直接关掉保护，这是一个严重的安全漏洞。

**修复方案：**
- 当密码保护当前为**开启**状态、用户点击"关闭密码保护"时，先弹出密码验证弹窗（复用已有的 `requirePassword()` 机制）
- 验证通过后才执行关闭操作
- 当密码保护当前为**关闭**状态、用户点击"开启密码保护"时，不需要验证（密码已设置，开启保护不降低安全性）

**修改文件：**
- `js/app.js`
  - `togglePassword()` → 判断当前状态，关闭时走 `requirePassword()` 验证
  - 新增 `_performToggle(enabled)` → 实际执行切换 + CloudBase 同步
