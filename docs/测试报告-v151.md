## 修改内容
1. **防止日志脱敏字符串污染数据**
   - 新增 `_isValidPasswordHash(hash)`：检测并拒绝 `"len=N,head=..."` 这类被污染的假 hash。
   - `_getLocalDataPackage()` 在组装推送数据前过滤无效 hash。
   - `_applyDataPackage()` 不再把无效 hash 写入 localStorage。

2. **`pushToCloud()` 改用 `set` 覆盖写**
   - 移除 create/update/delete-recreate 复杂逻辑，统一使用 `docRef.set(docData)`。
   - `set` 行为：文档不存在则创建，存在则完全覆盖，避免主键冲突 `DATABASE_REQUEST_FAILED`。

3. **严格错误处理与验证**
   - 新增 `_isCloudBaseError(res)`：检查 `res.code / errCode / error / Error`。
   - `set` 返回包含错误码时直接抛错，不再继续假成功流程。
   - 推送后验证通过 `where` 查询拉取，并比对 `updatedAt` 时间戳，确保不是旧数据。
   - 额外比对云端 `passwordHash` 与本地是否一致，不一致视为失败。
   - 验证未通过时 `pushToCloud()` 返回 `false` 并记录错误状态。

4. **版本统一升级到 v151**
   - `index.html`、`service-worker.js`、`js/storage.js`、`tests/smoke_test.js`。
