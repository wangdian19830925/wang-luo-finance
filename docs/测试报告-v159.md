# 测试报告 v159

- **版本号**：v159

- **变更类型**：UI/UX 细节调整 + 宏观趋势布局优化

## 变更内容

### 1. 额外收支按钮与图例位置精调

**需求**：
- 「额外收支」按钮放到「房贷处理方式」右侧同一行，与单选项视觉对齐。
- 按钮长度缩短一半，靠左。
- 有数据时，图例在按钮右侧水平排列；过多时隐藏最旧记录。

**实现**：
- `index.html` 保持按钮与图例在同一容器内。
- `css/style.css` 调整 `.retirement-mortgage-right` 为 `flex: 0 0 auto`、`justify-content: flex-start`，使按钮紧贴 radio group 右侧；`.retirement-extra-btn` 改为固定宽度（max-width: 96px）。
- `_updateExtraTransactionBadge()` 按年份从新到旧排序，配合 `overflow:hidden` 隐藏最旧项。

**修改文件**：`css/style.css`、`js/app.js`

### 2. 退休曲线刻度颜色统一

**需求**：CPI / 年化收益曲线图的刻度颜色改为与「可投资产余额趋势」一致（#94a3b8），并移除曲线下方和左侧的「年份」「CPI」「年化收益」轴标题。

**实现**：
- `_renderRetirementCurveEditor()` 中 X/Y 轴刻度文字统一添加 `fill="#94a3b8"`。
- 删除 SVG 底部的「年份」标签和左侧的「CPI (%)」「年化收益 (%)」标签。

**修改文件**：`js/app.js`

### 3. 宏观趋势「应用」按钮拆分

**需求**：删除「预测参数」卡片；分别在「CPI 走势」和「利率走势」卡片右上角放置「应用」按钮，分别将 CPI / 利率机构预测应用到退休计算。

**实现**：
- `index.html` 删除「预测参数」卡片；CPI / 利率卡片 header 拆分为左侧标题区 + 右侧「应用」按钮，ID 分别为 `applyMacroCpiBtn`（data-type="inflation"）和 `applyMacroRateBtn`（data-type="investmentReturn"）。
- `css/style.css` 新增 `.macro-trends-card-header-left` 和 `.macro-trends-apply-btn` 样式。
- `_bindMacroTrendsActions()` 绑定两个新按钮；`applyMacroCurveToRetirement(type)` 支持按类型单独应用，并给出对应 toast 提示。

**修改文件**：`index.html`、`css/style.css`、`js/app.js`

### 4. 每日新闻卡片置顶并修复无数据问题

**需求**：每日新闻卡片放到宏观趋势页最上方，并修复没有数据的问题。

**实现**：
- `index.html` 将「每日新闻」卡片移至 summary cards 之前的最顶部。
- `_fetchMacroNews()` 移除 `rss2json` 的 `count` 参数（该参数需要 API key，会导致返回错误）。

**修改文件**：`index.html`、`js/app.js`
